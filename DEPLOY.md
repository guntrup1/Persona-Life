# Деплой Life OS на Render.com (бесплатно)

Пошаговая инструкция по размещению приложения на бесплатном сервере.

---

## Что понадобится

- Аккаунт на [GitHub](https://github.com) (бесплатно)
- Аккаунт на [Render.com](https://render.com) (бесплатно)
- Строка подключения MongoDB Atlas (у тебя уже есть — `MONGODB_URI`)

---

## Шаг 1: Создать репозиторий на GitHub

1. Зайди на https://github.com/new
2. Заполни:
   - **Repository name**: `life-os` (или любое другое название)
   - **Visibility**: выбери **Private** (код будет виден только тебе)
   - **НЕ ставь** галочки на "Add a README", "Add .gitignore", "Choose a license" — репозиторий должен быть пустым
3. Нажми **"Create repository"**
4. Скопируй URL репозитория — он будет выглядеть так:
   ```
   https://github.com/ТВОЙ-ЮЗЕРНЕЙМ/life-os.git
   ```

---

## Шаг 2: Загрузить код на GitHub

### Вариант А: Через Replit (проще)

1. В Replit открой свой проект
2. Нажми на вкладку **Git** (иконка ветки) на левой панели
3. Если GitHub не подключён — нажми "Connect to GitHub" и авторизуйся
4. Выбери свой репозиторий `life-os`
5. Нажми **"Commit & Push"**

### Вариант Б: Через терминал (если есть Git на компьютере)

Открой терминал Replit (Shell) и выполни команды:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ТВОЙ-ЮЗЕРНЕЙМ/life-os.git
git push -u origin main
```

После этого зайди на GitHub — в репозитории должны появиться все файлы проекта.

---

## Шаг 3: Создать аккаунт на Render.com

1. Зайди на https://render.com
2. Нажми **"Get Started for Free"**
3. Выбери **"Sign in with GitHub"** — это самый удобный вариант, потому что Render сразу получит доступ к твоим репозиториям
4. Разреши доступ к репозиториям (можно выбрать только `life-os`)

---

## Шаг 4: Создать Web Service на Render

1. На главной странице Render нажми **"New +"** → **"Web Service"**
2. Выбери **"Build and deploy from a Git repository"** → **"Next"**
3. Найди свой репозиторий `life-os` и нажми **"Connect"**
4. Заполни настройки:

| Поле | Значение |
|------|----------|
| **Name** | `life-os` (или любое) |
| **Region** | `Frankfurt (EU Central)` (ближе к тебе — быстрее) |
| **Branch** | `main` |
| **Runtime** | `Docker` |
| **Instance Type** | **Free** |

5. Нажми **"Create Web Service"**

---

## Шаг 5: Настроить переменные окружения

Render начнёт сборку, но приложение пока не запустится — нужно добавить переменные.

1. В панели своего сервиса на Render перейди в раздел **"Environment"**
2. Нажми **"Add Environment Variable"** и добавь:

| Key | Value |
|-----|-------|
| `MONGODB_URI` | Твоя строка подключения MongoDB Atlas (начинается с `mongodb+srv://...`) |
| `SESSION_SECRET` | Любая длинная случайная строка (например: `my-super-secret-key-12345`) |
| `PORT` | `10000` |
| `NODE_ENV` | `production` |

3. Нажми **"Save Changes"**
4. Render автоматически пересоберёт и перезапустит приложение

---

## Шаг 6: Проверить работу

1. Подожди 3–5 минут, пока Render соберёт Docker-образ и запустит приложение
2. В панели сервиса вверху будет ссылка вида:
   ```
   https://life-os-xxxx.onrender.com
   ```
3. Нажми на неё — должна открыться страница входа
4. Войди со своим логином и паролем

---

## Шаг 7: Настроить MongoDB Atlas (важно!)

MongoDB Atlas по умолчанию блокирует подключения с неизвестных IP-адресов. Нужно разрешить подключение с Render.

1. Зайди в https://cloud.mongodb.com
2. Перейди в свой проект → **"Network Access"** (в левом меню)
3. Нажми **"Add IP Address"**
4. Нажми **"Allow Access from Anywhere"** (это добавит `0.0.0.0/0`)
   - Это безопасно, потому что подключение всё равно требует логин и пароль
5. Нажми **"Confirm"**

Если этого не сделать, приложение не сможет подключиться к базе данных.

---

## Обновление приложения

Каждый раз, когда ты пушишь изменения в GitHub (`git push`), Render автоматически:
1. Скачает новый код
2. Пересоберёт Docker-образ
3. Перезапустит приложение

Это занимает 3–5 минут.

---

## Важные ограничения бесплатного плана Render

- **Сервер засыпает** через 15 минут без активности. Первый запрос после сна будет загружаться 30–60 секунд (cold start)
- **750 часов** бесплатной работы в месяц (хватает на один сервис 24/7)
- Нет кастомных доменов на бесплатном плане

---

## Если что-то не работает

### Приложение не запускается
- Проверь логи в Render: панель сервиса → вкладка **"Logs"**
- Убедись, что все переменные окружения заданы правильно

### Ошибка подключения к MongoDB
- Проверь, что IP `0.0.0.0/0` добавлен в Network Access в MongoDB Atlas
- Проверь, что `MONGODB_URI` указан правильно (без лишних пробелов)

### Страница не загружается (504 Gateway Timeout)
- Подожди 1–2 минуты — возможно, сервер ещё стартует после сна

## Переменные окружения — справочник

| Переменная | Обязательная | Описание |
|-----------|-------------|----------|
| `MONGODB_URI` | Да | Строка подключения к MongoDB Atlas |
| `SESSION_SECRET` | Рекомендуется | Секретный ключ для шифрования сессий |
| `PORT` | Да (на Render) | Порт сервера. На Render = `10000` |
| `NODE_ENV` | Да | Режим работы. Для продакшена = `production` |
