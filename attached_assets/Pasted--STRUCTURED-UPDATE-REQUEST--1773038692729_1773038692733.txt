ИЗМЕНЕНИЯ В ПРИЛОЖЕНИИ (STRUCTURED UPDATE REQUEST)

Внеси следующие изменения в существующее приложение Gamified Life OS.

Все изменения должны быть реализованы без нарушения уже работающих функций.

1. ИЗМЕНЕНИЯ ГЛАВНОГО ХАБА
1.1 Отображение торговых сессий

Из главного хаба необходимо убрать расписание всех торговых сессий.

Вместо этого должно отображаться только текущее состояние:

Показывать:

текущую активную сессию

текущее время пользователя

Пример отображения:

Текущая сессия: London
Время: 10:32 (UTC+1)

Сессия должна обновляться автоматически в реальном времени.

2. НОВАЯ СТРУКТУРА ГЛАВНОГО ХАБА

Перераспредели элементы главного хаба следующим образом.

2.1 Левая часть хаба

В левой части должны находиться:

Блок аватара

Иконка со смайликом персонажа.

В этом же блоке должны отображаться:

текущая торговая сессия

уведомления о новостях (если есть)

2.2 Правая часть хаба

В правой части хаба должны находиться списки задач и прогресса.

В следующем порядке:

1️⃣ Задачи недели
2️⃣ Прогресс недели
3️⃣ Прогресс дня
4️⃣ Задачи дня

2.3 Сворачиваемые блоки

Каждый блок должен быть сворачиваемым.

Пользователь должен иметь возможность:

свернуть блок

развернуть блок

Это нужно чтобы, например, скрыть задачи недели, если они мешают.

2.4 Нижняя часть хаба

Ниже этих блоков можно оставить остальные элементы интерфейса:

статистику

дополнительные панели

3. ИЗМЕНЕНИЯ В ШАБЛОНЕ РУТИНЫ

В разделе Шаблон Рутины необходимо добавить новую кнопку:

"Обновить рутину"

3.1 Логика кнопки "Обновить рутину"

Кнопка должна выполнять следующую логику:

Сравнить список задач в шаблоне рутины

Сравнить список задач текущего дня

После этого:

Если в шаблоне есть задачи, которых нет в сегодняшнем списке,
они должны добавляться в задачи сегодняшнего дня.

3.2 Важное условие

Если задача уже существует в списке задач на сегодня:

она не должна добавляться повторно.

Это нужно для предотвращения дублирования задач.

3.3 Синхронизация с главным хабом

После обновления рутины:

список задач дня должен обновиться

главный хаб должен автоматически обновить отображение задач

4. АВТОРИЗАЦИЯ ПОЛЬЗОВАТЕЛЯ

Добавить систему авторизации через Google аккаунт.

Использовать вход через аккаунт Google.

4.1 Функции авторизации

Пользователь должен иметь возможность:

войти через Google

использовать один и тот же аккаунт на разных устройствах

Например:

телефон

компьютер

4.2 Синхронизация данных

После входа:

данные пользователя должны автоматически синхронизироваться:

задачи

рутина

цели

статистика

заметки

4.3 Сессия входа

После входа:

пользователь должен оставаться авторизованным.

Повторный вход не требуется.

Сессия должна завершаться только если пользователь:

нажал Выйти из профиля.

5. ИЗМЕНЕНИЕ ЛОГИКИ XP

Измени систему XP для задач.

5.1 Возможность ручной настройки XP

При создании задачи пользователь должен иметь возможность сам задавать XP.

Это касается:

задач на день

задач недели

задач месяца

задач года

5.2 Значение XP

XP должно задаваться пользователем вручную.

Пример:

Task: Изучить стратегию
XP: 40 XP

5.3 Использование XP

XP должно использоваться для расчёта:

прогресса дня

прогресса недели

прогресса месяца

прогресса года

5.4 Важное условие

Если задача удалена:

XP должно автоматически пересчитываться.

6. СИНХРОНИЗАЦИЯ ДАННЫХ

После выполнения любого действия:

добавление задачи

удаление задачи

обновление рутины

данные должны сразу обновляться в:

главном хабе

статистике

прогрессе

Без необходимости перезагрузки страницы.

7. АНИМАЦИИ

Все изменения интерфейса должны сопровождаться:

плавными анимациями

мягкими переходами

Особенно:

сворачивание блоков

обновление задач

изменение XP

изменение прогресса

РЕЗУЛЬТАТ

После внедрения этих изменений приложение должно:

иметь более удобный главный хаб

корректно обновлять рутину

не создавать дубли задач

поддерживать синхронизацию через Google аккаунт

позволять пользователю самому управлять XP задач
